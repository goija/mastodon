Systeem Architectuur: Omni-Bridge & Helix Dashboard

De koppeling tussen het Python script en de HTML frontend vormt een asynchrone client-server architectuur, verbonden via WebSockets. Dit scheidt de zware wiskundige berekeningen van de DOM-rendering.

1. De Backend: VORTEX_Bridge.py

Dit is het kloppende hart van het systeem. Het Python-script draait lokaal een asynchrone WebSocket server op poort 8765. Het berekent continu de telemetrie voor de 234 nodes van de ELS-matrix: de modulaire zwaartekracht naar het $(9,9)$ nulpunt, de schroeflijn-hoeken en de wiskundig zuivere 13-limiet frequenties. Deze ruwe data wordt gecomprimeerd in een JSON-payload en weggeschoten in een ritme van exact $13.75$ Hz.

2. De Frontend: vortex_helix_dashboard.html

Dit is de tactische interface. Geschreven in Vue.js, "luistert" dit HTML-bestand naar de poort van de Python server. Zodra de JSON-payload binnenkomt, werkt Vue de DOM reactief bij zonder de browser te blokkeren. Hier worden de inkomende ELS-sprongen geprojecteerd op het $13 \times 18$ visuele grid. Tevens pakt de WebAudio API de inkomende 13-limiet signalen op en synthetiseert deze direct tot hoorbare, zuivere geluidsgolven via je lokale speakers.
